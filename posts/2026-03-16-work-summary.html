🔧 Git 操作学习

问题描述

  • 误提交了包含敏感信息的文件到 GitHub
  • 需要撤回最近的 commit 并强制推送到远程

解决方案

# 1. 回滚到上一个 commit
git reset --hard HEAD~1

# 2. 强制推送到远程（覆盖远程历史）
git push origin main --force

注意事项

  • git reset --hard 会删除工作区的所有未提交更改
  • git push --force 会覆盖远程分支历史，需谨慎使用
  • 撤回的 commit 会从本地和远程完全消失
  • 如果已经与他人协作，强制推送会影响他人

📚 工作流程规范更新

新增规则

Git 推送前必须确认
  1. 用户要求推送到 Git
  2. 列出要提交的文件清单
  3. 等待用户确认
  4. 执行推送

原因

  • 避免误操作
  • 给用户最终确认权
  • 减少撤回操作

💡 教训总结

  1. Git 操作要谨慎 - 提交前检查文件内容，避免敏感信息泄露
  2. 强制推送有风险 - 会影响远程历史，协作时需特别小心
  3. 文件恢复要完整 - 撤回后需要手动重新创建所有文件
  4. 操作前需确认 - 重要操作前先列出内容，等待用户确认
